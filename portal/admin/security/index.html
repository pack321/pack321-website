Pack 321 Portal Architecture

Security Controls

Required security rules and settings for the Pack 321 Portal once it moves behind a secured backend/API.

My DayDashboardUsersRolesSecurityAudit LogSettings

Authentication

  • Google OAuth only
  • No local passwords
  • No password storage
  • No forgot password workflow
  • Require active user record

Authorization

  • Match Google email to Users table
  • Load assigned roles
  • Load RolePermissions
  • Allow or deny by permission key
  • Enforce on server/API layer

Settings

  • Session timeout
  • Inactive user handling
  • Last login tracking
  • Audit logging enabled
  • Export logging enabled
Important: frontend hiding is not security. Every read/write API function must check the signed-in user and permission key before returning or changing data.
