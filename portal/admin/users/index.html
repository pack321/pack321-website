AccessRequestsrequestId, name, email, familyName, scoutNames, requestedRole, notes, status, reviewedBy, reviewedAt, createdAtTracks request/approval workflow for unauthenticated-but-not-authorized users.

Approval Flow

  1. Admin reviews pending request.
  2. Admin links request to existing family/scout or creates records.
  3. Admin assigns one or more roles.
  4. System creates or activates user record.
  5. System logs approval in AuditLog.
