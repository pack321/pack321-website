Pack 321 Portal Architecture

Roles & Permissions

Role-based access design for parents, den leaders, treasurer, committee chair, webmaster, and admin users.

DashboardUsersRolesSecurityAudit LogSettings
Recommended RoleDefault BehaviorNotes
ParentOwn family onlyNo den-wide, finance-wide, or admin access.
Den LeaderAssigned den onlyNo financial access unless another role is assigned.
CubmasterOperational Pack visibilityCan be expanded by assigned permissions.
TreasurerFull finance accessCan view reports, create/edit transactions, and export finance reports.
Committee ChairFull operational/report accessDoes not manage system security unless also Admin.
WebmasterContent and website toolsShould not imply finance or system security access.
AdminFull system/security accessManages users, roles, settings, and security controls.

Role Tables

  • Roles: roleId, roleName, description, active
  • Permissions: permissionId, permissionKey, description, category
  • RolePermissions: rolePermissionId, roleId, permissionId, allowed
