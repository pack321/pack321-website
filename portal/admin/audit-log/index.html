Audit Log

Audit design for security-sensitive actions, exports, financial changes, access changes, and settings updates.

My DayDashboardUsersRolesSecurityAudit LogSettings
FieldDescription
auditIdUnique audit event id.
timestampServer timestamp for the action.
userIdUser who performed the action.
actionUser created, role assigned, transaction edited, report exported, etc.
moduleAdmin, finance, family, scout, event, settings.
tableName / recordIdChanged record location.
oldValue / newValueBefore/after values for review.
ipAddress / userAgentRequest metadata for security review.

Log These Actions

  • User created or deactivated
  • Role assigned or removed
  • Permission changed
  • Family/scout record changed
  • Financial transaction created/edited
  • Correction request approved/denied
  • Report exported
  • Settings changed

Audit UI

  • Search by user/action/module
  • Filter by date range
  • View before/after changes
  • Export logging should also create an audit event
